7.2. Too many open HTTP connections

When using HttpSender, there are plenty of short-lived connections. When your test uses a lot of threads, you migth quickly run into the limits of the underlying system. You can either increase the number of connections, or on Linux system, you can make sure that the connections are reused like follows: 

   $ echo "1" >/proc/sys/net/ipv4/tcp_tw_reuse
   $ echo "1" >/proc/sys/net/ipv4/tcp_tw_recycle
